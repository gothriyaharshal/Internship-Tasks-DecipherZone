In Java, switch case is a conditional control statement used when you want to execute one block of code from many options
 based on a single value.

Syntax (traditional switch)
switch(expression)
{
    case value1:
        // code
        break;

    case value2:
        // code
        break;

    default:
        // code
}
Example:
import java.util.Scanner;

class SwitchDemo
{
    public static void main(String[] args)
    {
        Scanner sc = new Scanner(System.in);

        System.out.println("Enter day number (1-7)");
        int day = sc.nextInt();

        switch(day)
        {
            case 1:
                System.out.println("Monday");
                break;

            case 2:
                System.out.println("Tuesday");
                break;

            case 3:
                System.out.println("Wednesday");
                break;

            case 4:
                System.out.println("Thursday");
                break;

            case 5:
                System.out.println("Friday");
                break;

            case 6:
                System.out.println("Saturday");
                break;

            case 7:
                System.out.println("Sunday");
                break;

            default:
                System.out.println("Invalid day");
        }
    }
}
How it works internally:

Suppose input = 3

switch(3)

case 1 ❌
case 2 ❌
case 3 ✅
    print Wednesday
    break

break stops execution and comes out of switch.

What happens if we remove break?
int x = 2;

switch(x)
{
    case 1:
        System.out.println("One");

    case 2:
        System.out.println("Two");

    case 3:
        System.out.println("Three");
}

Output:

Two
Three
4

Because after finding case 2, Java continues running next cases (fall-through) until it finds break or switch ends.

Multiple cases same code
int day = 6;

switch(day)
{
    case 6:
    case 7:
        System.out.println("Weekend");
        break;

    default:
        System.out.println("Working day");
}

Output:

Weekend
Modern Java switch (Java 14+)

No need for break:

int day = 2;

switch(day)
{
    case 1 -> System.out.println("Monday");
    case 2 -> System.out.println("Tuesday");
    case 3 -> System.out.println("Wednesday");
    default -> System.out.println("Invalid");
}

✅ int

✅ enum

Not allowed:

❌ float
❌ double
❌ boolean

In latest Java (Java 14+), switch has evolved. Earlier switch was mainly a statement with case, break, and fall-through behavior.
 Now Java introduced switch expressions which are safer and can return values.

Let's understand deeply.

1. Old switch vs Modern switch
Old style (switch statement)
int day = 2;

switch(day)
{
    case 1:
        System.out.println("Monday");
        break;

    case 2:
        System.out.println("Tuesday");
        break;

    default:
        System.out.println("Invalid");
}

Problem:

Need break
Forgetting break causes fall-through
2. New switch expression (-> syntax)

Modern Java:

int day = 2;

switch(day)
{
    case 1 -> System.out.println("Monday");
    case 2 -> System.out.println("Tuesday");
    default -> System.out.println("Invalid");
}

Here:

case value -> statement

means:

"If this case matches, execute this code only."

No break needed.

3. Why break is not required now?

Because -> automatically prevents fall-through.

Example:

int x = 2;

switch(x)
{
    case 1 -> System.out.println("One");
    case 2 -> System.out.println("Two");
    case 3 -> System.out.println("Three");
}

Output:

Two

After executing case 2, switch finishes.

4. Switch can return a value now

This is the biggest change.

Old way:

int day = 3;
String result;

switch(day)
{
    case 1:
        result = "Monday";
        break;

    case 2:
        result = "Tuesday";
        break;

    default:
        result = "Invalid";
}

System.out.println(result);

Modern way:

int day = 3;

String result = switch(day)
{
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    default -> "Invalid";
};

System.out.println(result);

Output:

Wednesday

Here switch itself gives a value.

5. How yield works

If a case has multiple statements:

int marks = 85;

String grade = switch(marks)
{
    case 90, 100:
        yield "A";

    case 70, 80, 85:
        yield "B";

    default:
        yield "C";
};

System.out.println(grade);

Output:

B

yield returns the value from switch.

Think:

yield = return value from switch block
6. Multiple values in one case

Old:

case 1:
case 2:
case 3:
    System.out.println("Small number");

Modern:

int num = 2;

switch(num)
{
    case 1,2,3 -> System.out.println("Small number");
    case 4,5,6 -> System.out.println("Medium");
    default -> System.out.println("Large");
}

Cleaner.

7. Switch with String

Example:

String role = "ADMIN";

switch(role)
{
    case "ADMIN" -> System.out.println("Full access");
    case "USER" -> System.out.println("Limited access");
    default -> System.out.println("Unknown");
}
8. Switch with enum
enum Day
{
    MONDAY, TUESDAY, WEDNESDAY
}

class Test
{
    public static void main(String args[])
    {
        Day d = Day.MONDAY;

        switch(d)
        {
            case MONDAY -> System.out.println("Start");
            case TUESDAY -> System.out.println("Second day");
            case WEDNESDAY -> System.out.println("Third day");
        }
    }
}
9. Important: case matching rules

For switch:

switch(expression)

expression value is compared with case values.

Example:

int x = 10;

switch(x)
{
    case 10 -> System.out.println("Ten");
}

Internally:

x == 10 ?

If true → execute.

10. Pattern matching switch (newer Java versions)

Modern Java also supports patterns (Java 21+).

Example:

Object obj = "Hello";

switch(obj)
{
    case String s -> System.out.println(s.length());

    case Integer i -> System.out.println(i * 2);

    default -> System.out.println("Other");
}

Output:

5

Here switch checks type also.

When to use switch?

Use switch when:

✅ Checking one variable against many fixed values

Example:

Menu system
Days
Months
User roles
Commands

Use if-else when:

✅ Conditions are complex

Example:

if(age > 18 && salary > 50000)
Simple mental model:

Old switch:

find case
↓
execute
↓
break required

Modern switch:

find case
↓
execute
↓
automatically exit
↓
can return value

Modern Java switch is closer to a value-producing expression rather than just a control statement.

Package in Java (Deep Dive)

A package in Java is a mechanism used to group related classes, interfaces, enums, etc. together. It works like a folder/directory structure and helps in:

Organizing code
Avoiding name conflicts
Providing access control
Making code reusable

Think of a package as a container of classes.

Example:

com.company.project
        |
        |---- User.java
        |---- Login.java
        |---- Database.java

Here all these classes belong to the same package.

1. Why do we need packages?

Suppose you create two classes:

Student.java

and another developer also creates:

Student.java

Both have the same name.

Without packages:

Student
Student

Java cannot know which one you mean.

With packages:

college.Student
school.Student

Now they are different.

2. Types of Packages

Java has two types:

1. Built-in packages

Provided by Java.

Examples:

java.lang
java.util
java.io
java.sql

Example:

import java.util.Scanner;

class Test
{
    public static void main(String args[])
    {
        Scanner sc = new Scanner(System.in);
    }
}

Here:

Scanner

class exists inside:

java.util

package.

2. User-defined packages

Created by programmers.

Example:

mypackage
3. Creating your own package

Suppose your project:

Project
 |
 |--- Student.java
 |
 |--- Test.java

Student.java:

package mypackage;

public class Student
{
    public void display()
    {
        System.out.println("Student class");
    }
}

Important:

The package statement must be the first line.

Correct:

package mypackage;

class Student
{

}

Wrong:

import java.util.*;

package mypackage; // error
4. Using another package class

Another file:

import mypackage.Student;

class Test
{
    public static void main(String args[])
    {
        Student s = new Student();

        s.display();
    }
}

Output:

Student class

14. Internal working of package

When you write:

import java.util.Scanner;

Compiler does:

Searches classpath
Finds:
java/util/Scanner.class
Loads bytecode
Allows usage

Import does not copy code.

It only tells compiler where to find class.
